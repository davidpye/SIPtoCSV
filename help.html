SIP API CSV Exporter - Help

What is this Application for?

This Application allows Hub Partners on the Unlocking Our Sound Heritage project to access the data they submitted to the British Library via the SIP tool, by collecting the data from the BL's web APIs and converting it into standard CSV format. (An API is a way for one web-based application to speak to another. Allowing this application to collect the data for you, and automate the conversion process.)

How to use this Application:

First, make sure you are using a computer which has access to the BL's VPN for the UOSH project.
If you are not connected to the VPN, this application will not work.

Once connected to the VPN, also make sure you are logged into the SIP tool by visiting https://avsip.ad.bl.uk/ and inputting your user credentials. Once logged in, you can close the SIP tool and start to request your data using this Application.

Start by enterring a few Call Numbers into the text box at the top of the page. You can copy and paste a list of Call Numbers from your tracking spreadsheets. They can be comma, space, or new line separated, but must match the Call Numbers stored on the SIP tool perfectly.
Click the Submit button at the bottom of the page to collect the data.

The app works by running searches for the Call Numbers on the SIP tool to access their SIP ID numbers, which it will then use to access the Recording/Logical and Process Metadata. It will also grab the SAMI IDs, and run a separate pair of searches on SAMI to access the Cataloguing Metadata at both the Product, and Recording level. All this Data is then combined into a CSV file which will automatically download onto your PC when the process is finished.

You will notice a few text boxes for inputting additional information, these are mostly prefixes, and aren't required to run the Application.
The data generated is designed to be imported into AtoM Archival Management software, and so these prefixes are aimed to aid the import process.
More information on the CSV import process can be found here.

Please note:
Currently there are some minor whitespacing issues with the generated CSV file, which make the files display incorrectly if first opened in Microsoft Excel, and will cause errors when importing directly into AtoM.
To get around this issue, first open the CSV in Google Drive, where the whitespacing is (for some reason) parsed correctly, then export a CSV from there. This will also give you a chance to check over the structure of the data and bulk fill any additional data you may wish to store.
